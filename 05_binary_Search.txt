1. BINARY SEARCH PATTERN

---

Binary Search is used when:

* Array is sorted
* Searching for an element or index
* Search space can be divided into two halves
* Time complexity requirement is O(log n)

Basic Idea:

Divide the search space in half each iteration.

left  -> start index
right -> end index

mid = left + (right-left)/2

Compare nums[mid] with target and shrink the search space.

---

2. CLASSIC BINARY SEARCH

---

Problem:
Given a sorted array nums and a target value,
return the index if target is found,
otherwise return -1.

Example:

nums = [-1,0,3,5,9,12]
target = 9

Output:
4

---

## ALGORITHM

1. Initialize left = 0, right = n-1
2. Compute mid
3. Compare nums[mid] with target

if nums[mid] == target
return mid

if nums[mid] < target
search right half

if nums[mid] > target
search left half

---

## C++ CODE

int search(vector<int>& nums, int target){

```
int left = 0;
int right = nums.size() - 1;

while(left <= right){

    int mid = left + (right-left)/2;

    if(nums[mid] == target)
        return mid;

    else if(nums[mid] < target)
        left = mid + 1;

    else
        right = mid - 1;
}

return -1;
```

}

Time Complexity: O(log n)
Space Complexity: O(1)

3. SEARCH IN ROTATED SORTED ARRAY

---

Problem:
Array is sorted but rotated.

Example:

nums = [4,5,6,7,0,1,2]
target = 0

Output:
4

---

## KEY IDEA

Even after rotation, one half of the array
is always sorted.

Check which half is sorted:

if nums[left] <= nums[mid]
left half is sorted

else
right half is sorted

Then decide whether the target lies
inside that sorted half.

---

## C++ CODE

int search(vector<int>& nums, int target){

```
int left = 0;
int right = nums.size()-1;

while(left <= right){

    int mid = left + (right-left)/2;

    if(nums[mid] == target)
        return mid;

    if(nums[left] <= nums[mid]){

        if(target >= nums[left] && target < nums[mid])
            right = mid - 1;
        else
            left = mid + 1;
    }

    else{

        if(target > nums[mid] && target <= nums[right])
            left = mid + 1;
        else
            right = mid - 1;
    }
}

return -1;
```

}

Time Complexity: O(log n)

4. FIND PEAK ELEMENT

---

Problem:
Find an index where the element is greater
than its neighbors.

Example:

nums = [1,2,3,1]

Output:
2

---

## KEY OBSERVATION

If nums[mid] < nums[mid+1]
peak must exist on the right side.

If nums[mid] > nums[mid+1]
peak exists on the left side.

Binary search works because the array
always contains a peak.

---

## C++ CODE

int findPeakElement(vector<int>& nums){

```
int left = 0;
int right = nums.size()-1;

while(left < right){

    int mid = left + (right-left)/2;

    if(nums[mid] < nums[mid+1])
        left = mid + 1;
    else
        right = mid;
}

return left;
```

}

Time Complexity: O(log n)
Space Complexity: O(1)

---

## IMPORTANT BINARY SEARCH RULES

1. Always calculate mid as:

mid = left + (right-left)/2

to avoid integer overflow.

2. Loop conditions:

while(left <= right) -> exact search

while(left < right) -> search space narrowing problems

3. Binary search works when:

* array is sorted
* search space can be divided
